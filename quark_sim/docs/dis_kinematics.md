# Neutral-current inclusive DIS kinematics

This document describes the reusable kinematics core for

```text
e⁻(k) + p(P) -> e⁻(k') + X
```

It computes Lorentz invariants for an inclusive neutral-current event. It does
not calculate a cross section or construct the hadronic state `X`.

## Units and constants

The core uses GeV-based natural units with `c = 1`:

- energies, masses, and momentum components are in GeV;
- squared invariants are in GeV²;
- `x` and `y` are dimensionless.

The particle masses are:

```text
m_e = 0.00051099895000 GeV
m_p = 0.93827208816 GeV
```

They are the electron and proton masses listed in the
[Particle Data Group 2024 physical-constants table](https://pdg.lbl.gov/2024/reviews/constants_atomic_and_related.html).

## Four-vector and metric convention

`FourVector` stores `(E, p_x, p_y, p_z)` and uses

```text
g = diag(+1, -1, -1, -1).
```

Therefore,

```text
a·b = a.E b.E - a.p_x b.p_x - a.p_y b.p_y - a.p_z b.p_z
p²  = E² - |p⃗|².
```

The implementation supports construction, checked finite-value construction,
addition, subtraction, the Minkowski dot product, invariant mass squared,
spatial momentum magnitude, and transverse momentum.

## Collider-frame convention

The provided beam helpers construct counter-propagating, on-shell beams:

```text
k = (E_e, 0, 0, +sqrt(E_e² - m_e²))
P = (E_p, 0, 0, -sqrt(E_p² - m_p²)).
```

The incoming electron travels along `+z`; the proton travels along `-z`.
The scattered-electron angle `theta` is measured from the incoming electron's
`+z` direction. The helper fixes its azimuth to zero:

```text
k' = (E_e', p' sin(theta), 0, p' cos(theta))
p' = sqrt(E_e'² - m_e²).
```

Inclusive scalar invariants do not depend on this azimuth choice.

No massless-beam approximation is used in these constructors. An energy below
the corresponding rest mass is rejected.

## Implemented equations

The exchanged-boson momentum and inclusive invariants are

```text
q  = k - k'
Q² = -q²
s  = (P + k)²
x  = Q² / (2 P·q)
y  = (P·q) / (P·k)
W² = (P + q)².
```

For an on-shell incoming proton, the implementation is numerically checked
against

```text
W² = m_p² + Q²(1/x - 1).
```

For on-shell incoming beams, another useful identity is

```text
Q² = x y (s - m_p² - m_e²).
```

## Validation and errors

The API returns the typed `DisError` enum and never clamps an unphysical value.
It rejects:

- NaN or infinite vector components, beam values, angles, or derived results;
- non-positive particle energies;
- beam energies below the relevant rest mass;
- incoming proton and incoming/outgoing electron vectors that are off their
  expected mass shells beyond a floating-point cancellation-aware tolerance;
- angles outside `[0, 180]` degrees;
- non-positive `2 P·q` or `P·k` denominators;
- `Q² <= 0`;
- `s <= 0`;
- `x <= 0` or `x >= 1`;
- `y <= 0` or `y >= 1`;
- `W² <= m_p²`, a minimal invariant-domain check below the proton mass;
- non-finite or incorrectly ordered analysis cuts.

`DisCuts` applies inclusive boundaries after event-level physical validation:

```text
q2_min <= Q² <= q2_max (when q2_max is present)
x_min  <= x  <= x_max
y_min  <= y  <= y_max
w2_min <= W².
```

A rejected event is not modified.

## CLI

From the `quark_sim` directory:

```bash
cargo run --release -- dis-kinematics \
  --electron-energy 27.5 \
  --proton-energy 920.0 \
  --scattered-electron-energy 15.0 \
  --theta-deg 20.0
```

Expected output:

```text
Neutral-current inclusive e⁻p DIS kinematics
s   = 101200.854031085386 GeV²
Q²  = 49.753587913075 GeV²
x   = 0.001043829650
y   = 0.470992917430
W²  = 47615.597612251062 GeV²
```

Subcommand help is available without initializing Candle, creating output
directories, training a model, or opening the GUI:

```bash
cargo run --release -- dis-kinematics --help
```

## Current limitations

- Only kinematics for inclusive neutral-current electron-proton scattering are
  represented. No electroweak matrix element is evaluated.
- There are no PDFs, structure functions, cross sections, radiative
  corrections, beam polarization, heavy-flavour scheme, or nuclear effects.
- The final hadronic system is represented only by its invariant mass; no
  hadrons, partons, or event record are produced.
- There is no random event generation, detector simulation, or acceptance
  model beyond explicit scalar cuts.
- Raw proton/electron `FourVector` inputs must already lie on their expected mass
  shells within a tolerance scaled for cancellation in `E² - |p⃗|²`. The core
  rejects off-shell vectors; it never projects or modifies them. The beam helpers
  are the preferred construction path.
- `W² > m_p²` is only the core's minimal invariant-domain guard. It is not by
  itself a resonance-exclusion or DIS-region selection; analyses must set an
  appropriate `w2_min` (and other cuts) for their scientific convention.
- The legacy Cornell neural-network visualization remains separate and is not
  used in this module.
